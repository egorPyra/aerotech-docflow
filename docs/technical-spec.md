# Aerotech Docflow — техническое задание MVP

## 1. Что это за система

Aerotech Docflow — локальный сервис для перевода бумажных документов в цифровой архив.

Он связывает:

- Planfix — хранит задачу, реквизиты документа, статус и ссылку на файл;
- EPSON DS-790WN — создаёт PDF документа;
- файловый сервер — хранит оригиналы PDF;
- Яндекс Диск — синхронизирует архив для удалённого доступа.

Сервис не является отдельной системой документооборота. Его задача — автоматизировать путь от скана до ссылки в Planfix.

```text
Planfix → сканирование → PDF → архив на сервере → ссылка в Planfix
```

---

## 2. Цель MVP

Сделать рабочий сервис, который:

1. получает из Planfix задачу на сканирование;
2. ждёт PDF, созданный сканером во входящей папке;
3. переименовывает и переносит PDF в архив;
4. записывает имя и путь к файлу обратно в Planfix.

В MVP оператор всё ещё запускает нужную предустановку на панели сканера.
Сервис автоматизирует всё после появления PDF: поиск файла, переименование, сохранение и обновление Planfix.

---

## 3. Как работает система

```text
Оператор создаёт задачу в Planfix
        ↓
Кнопка интерфейса запускает это ПО
        ↓
ПО отправляет команду сканеру
        ↓
Сканер создаёт PDF
        ↓
Сервис находит новый PDF
        ↓
Сервис формирует имя и переносит файл в архив
        ↓
Сервис возвращает в Planfix ссылку файла из Яндекс.Диск
```

### `main.py`

Главный файл приложения.

Содержит основной процесс обработки документа:

```text
получить задачу из Planfix
→ получить PDF через scanner.py
→ сохранить PDF через storage.py
→ завершить задачу через planfix.py
```

Основные функции:

- запуск сервиса;
- отслеживание процесса, главный мозг сервиса
- запуск обработки задачи;
- передача результата между модулями.

---

### `planfix.py`

Модуль для работы с Planfix API.

Основные функции:

- получить задачу, ожидающую сканирования;
- получить реквизиты документа из задачи;
- изменить статус задачи;
- записать в задачу имя файла, путь и ссылку;
- записать результат в архивную запись документа.

Подробнее в `docs/api-contract.md`.

---

### `scanner.py`

Модуль для работы со сканером через входящую папку.

Основные функции:

- общение со сканером;
- найти новый PDF во входящей папке;
- вернуть путь к найденному PDF;

Сканер сохраняет PDF в сетевую папку через свои предустановки.

---

### `storage.py`

Модуль для работы с файлами архива.

Основные функции:

- форматирование имени;
- переместить PDF из входящей папки в архив;
- вернуть итоговый путь к файлу.

## 5. Правила имени и хранения файла

Формат имени:

```text
DOCTYPE_DDMMYY_HHMM_NUMBER.pdf
```

Пример:

```text
НКЛ_090626_1457_2455B.pdf
```

Состав имени:

| Часть | Значение |
| --- | --- |
| `DOCTYPE` | Код типа документа из Planfix |
| `DDMMYY` | Дата сканирования |
| `HHMM` | Время сканирования |
| `NUMBER` | Номер документа из Planfix |

Недопустимые для Windows символы в номере документа заменяются на `_`.

Файл нельзя перезаписывать. Если итоговое имя уже занято, обработка завершается ошибкой.
